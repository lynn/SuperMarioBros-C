How a byte becomes a note

Each frame (1/60 s) the engine visits every channel and counts down the current note. When a counter reaches zero it pulls the next byte from that channel's stream. A “song” is one header — a tempo preset plus four stream offsets into a shared blob. Square 2 carries the melody and ends on a 00, and that terminator loops all four voices at once.

ChannelByte formatLength comes from…
Square 2 / Triangle 8x sets a rhythm (low 3 bits); plain even bytes are notes that ride it. 00 ends the song (Square 2) or rests (Triangle). rhythm bits → MusicLengthTbl[bits + tempo]
Square 1 / Noise one byte holds both — note = byte & 0x3E, the length code is bits d7 d6 d0. Noise loops back on its own 00. packed bits → the same table

Pitch is a table too: note/2 indexes NES period dividers, and Hz = 1789773 / (16·(divider+1)). The triangle writes the same divider but its hardware ticks half as fast — hence an octave lower. Square and noise timbres here are approximations (Web Audio oscillators and filtered noise); the note data is exact.
